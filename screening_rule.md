# Правило скрининга сайта клиники

Версия: 2026-04-16

Этот документ — обязательный скрипт для агента при проверке одного сайта.
Проверять строго по одному сайту за раз.

---

## Принцип работы

**Только факты. Никаких выводов и интерпретаций.**
Агент фиксирует что именно нашёл — цитату, ссылку, текст элемента.
Решение "проблема это или нет" принимает человек по зафиксированным фактам.

---

## Шаг 1. Страницы для обхода (без "угадывания" URL)

Перед проверкой собери список страниц для анализа. Работай в таком порядке:

1. **Обязательно открой главную страницу** (`/`).
2. **Собери URL контактов и записи из реальных источников сайта** (в приоритете):
   - `sitemap.xml` (и sitemap из `robots.txt`, если есть),
   - ссылки меню/хедера/футера на главной,
   - внутренние ссылки с текстами-ключами:
     - контакты: `контакты`, `contacts`, `contact`, `как добраться`,
     - запись: `запись`, `записаться`, `appointment`, `booking`, `online`.
3. **Открой найденные URL** и добавь их в список обхода.
4. **Собери все страницы с формами**:
   - из уже открытых страниц,
   - из `sitemap.xml` (если доступен) — открывай только страницы, где есть `<form` в HTML.
5. **Для страниц с формами из sitemap отдельно проверь признаки "записи" по контенту**:
   - текст/кнопки с корнем `запис` (`запись`, `записаться`),
   - `appointment`, `booking`, `online`,
   - формулировки типа `консультац`, `оставьте номер`, `обратный звонок`.
   - Если URL не содержит слова `запись`, но на странице есть форма + признаки записи — считай это найденной страницей записи.

### Fallback только если ничего не найдено

Если после шагов выше не найдено:
- ни одной страницы контактов — применяй fallback для контактов;
- ни одной страницы записи (включая кандидаты по форме/контенту) — применяй fallback для записи.

Fallback ограниченный (не более 3 URL на тип):
- контакты: `/contacts`, `/contact`, `/kontakty`
- запись: `/zapis`, `/appointment`, `/booking`
- не делать "широкий перебор" десятков путей.

Зафиксируй:
- какие URL были найдены через sitemap/навигацию,
- какие реально открылись (статус 200),
- какие вернули ошибку,
- применялся ли fallback.

## Шаг 2. Чеклист проверок

### 2.1 Сертификат безопасности

Проверь: открывается ли сайт по HTTPS без ошибки браузера.

**Если проблема:** укажи что именно — сертификат просрочен, самоподписан, сайт не открывается по HTTPS.

---

### 2.2 Формы на сайте

Найди **все** формы на всех посещённых страницах. Для каждой формы отдельно проверь:

**A. HTTPS передача данных**
Смотри на атрибут `action` формы.
- Если `action` начинается с `http://` — это проблема.
- Если `action` начинается с `https://`, относительный путь, или форма отправляется через JS на HTTPS-сайте — ок.

**Если проблема:** укажи страницу и значение атрибута `action`.

**B. Чекбокс согласия на обработку персональных данных**

Ищи `<input type="checkbox">` рядом с текстом про согласие / персональные данные / политику конфиденциальности.

Зафиксируй для каждой формы:
- Есть ли чекбокс? (да / нет)
- Если есть — предотмечен ли по умолчанию? (атрибут `checked` в HTML)
- Если нет чекбокса — есть ли хотя бы статичный текст со ссылкой на политику?

**Если проблема:** укажи страницу, название/ID формы, что именно найдено или отсутствует.

---

### 2.3 Корпоративная почта и SPF / DMARC

**Шаг А.** Найди контактный email на сайте (страница контактов, футер).

- Если почта на стороннем сервисе (`@gmail.com`, `@yandex.ru`, `@mail.ru`, `@bk.ru`, `@inbox.ru` и т.п.) — **проверку SPF/DMARC не проводить**, указать "н/п — почта на стороннем сервисе".
- Если почта на домене сайта — перейти к шагу Б.

**Шаг Б.** Проверь DNS-записи домена:
- SPF: `TXT`-запись домена содержит `v=spf1`?
- DMARC: `TXT`-запись `_dmarc.домен` существует и содержит `v=DMARC1`?

**Зафиксируй:** найденный email, наличие SPF (да/нет + значение), наличие DMARC (да/нет + значение).

---

### 2.4 Запрещённые слова и соцсети

Ищи на всех посещённых страницах следующие слова и ссылки — в тексте, в атрибутах ссылок (`href`), в атрибутах изображений (`alt`, `src`), в названиях иконок:

**Список для поиска:**
`meta`, `мета`,
`instagram`, `инстаграм`, `инстаграмм`,
`facebook`, `фейсбук`,
`whatsapp`, `вотсап`, `ватсап`,
`messenger`, `мессенджер фейсбук`,
`threads`,
`fb.com`, `instagram.com`, `wa.me`, `m.me`, `meta.com`

Для каждой находки зафиксируй:
- Страницу
- Точный текст или значение атрибута где найдено
- Визуально ли это видно пользователю (иконка, ссылка, текст) или только в коде

Наличие дисклеймера **не проверяй и не оценивай** — только фиксируй факт наличия упоминания и его контекст.

---

### 2.5 Политика конфиденциальности

Ищи на всех посещённых страницах:
- Ссылку с текстом типа: "политика конфиденциальности", "обработка персональных данных", "политика", "privacy"
- Ссылку ведущую на отдельную страницу или открывающую модальное окно

Зафиксируй: найдена ли ссылка, на какой странице, куда ведёт (URL или модал).

## Шаг 3. Формат вывода

Выдай результат в следующем формате:

```
Сайт: [домен]
Страницы проверены (найденные): [список URL со статусами]
Fallback-пробы URL: [список URL со статусами, если fallback применялся]

---

ДОСТУПНОСТЬ САЙТА
Статус: [ок / проверить / проблема]
PoC: [сколько найденных страниц открылись со статусом 200; если 0 из N — явно указать, что сайт недоступен в момент проверки]

---

СЕРТИФИКАТ
Статус: [ок / проблема]
PoC: [если проблема — что именно]

---

ФОРМЫ
[Для каждой формы:]
Страница: [URL]
Форма: [ID или назначение]
  HTTPS: [ок / проблема] — [action="..."]
  Чекбокс: [есть / нет / только текст]
  Предотмечен: [да / нет / н/п]
  PoC: [цитата из HTML если есть проблема]

---

ПОЧТА И SPF/DMARC
Email на сайте: [адрес]
Тип: [корпоративный / сторонний сервис]
SPF: [значение записи или "не найдено" или "н/п"]
DMARC: [значение записи или "не найдено" или "н/п"]

---

ЗАПРЕЩЁННЫЕ СЛОВА И СОЦСЕТИ
[Для каждой находки:]
Слово/домен: [что найдено]
Страница: [URL]
Контекст: [точный текст или значение атрибута]
Видимость: [видно пользователю / только в коде]

Если ничего не найдено: "не найдено"

---

ПОЛИТИКА КОНФИДЕНЦИАЛЬНОСТИ
Статус: [найдена / не найдена]
PoC: [страница где найдена + куда ведёт ссылка]
```

---

## Важные правила

1. **Не делай выводов.** Пиши только что нашёл. Не пиши "это нарушение" или "это ок" — только факты.
2. **Каждая форма — отдельно.** Не обобщай "все формы в порядке". Пройдись по каждой.
3. **Список запрещённых слов искать буквально** — не полагайся на память, ищи каждое слово из списка явно.
4. **Указывай что не удалось проверить** — если страница не открылась, форма не загрузилась, JS не выполнился — так и пиши.
5. **Одна проверка — один сайт.** Не смешивай данные разных сайтов.
